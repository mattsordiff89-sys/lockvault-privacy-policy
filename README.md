# LockVault Privacy Policy

**Effective date: July 12, 2026**

LockVault is an Android application created by Matthew Sordiff that allows users to protect selected apps using their device’s fingerprint, face recognition, PIN, pattern, or password.

This Privacy Policy explains what information LockVault accesses, how that information is used, and how it is protected.

## Information LockVault Accesses

### Installed app information

LockVault accesses the names and package identifiers of apps that can be launched on the device. This allows the user to choose which apps LockVault should protect.

The selected-app list is stored locally on the user’s device.

### Accessibility Service

LockVault uses the Android AccessibilityService API to detect when an app selected by the user is opened or when the active app changes.

This access is used only to:

- Detect when a protected app is opened.
- Display the LockVault authentication screen.
- Relock protected apps after the user leaves them.
- Help protect information when the Recent Apps screen is opened.

LockVault does not use Accessibility access to:

- Read messages or conversations.
- Read passwords or typed text.
- Read photos or files.
- Record the screen.
- Press buttons or perform actions on the user’s behalf.
- Collect the contents displayed inside other apps.

LockVault’s Accessibility Service is configured not to retrieve window contents.

Users must enable LockVault App Protection manually in Android Accessibility settings and may disable it at any time.

## Biometric and Device Authentication

LockVault uses Android’s system authentication interface.

Fingerprint, face, PIN, pattern, and password verification are handled by the Android operating system. LockVault does not receive, collect, store, or transmit biometric information or the user’s device password.

LockVault only receives confirmation from Android indicating whether authentication succeeded or failed.

## Data Collection

LockVault does not:

- Require an account.
- Collect names, addresses, phone numbers, or payment information.
- Collect messages, photos, passwords, or contacts.
- Collect precise location information.
- Use advertising or analytics services.
- Sell or share user information.
- Upload the selected-app list or protection settings to a server.

LockVault’s settings are stored locally on the device.

## Data Sharing

LockVault does not sell, rent, trade, or share user information with third parties.

## Data Storage and Deletion

Selected-app information, protection settings, and disclosure preferences are stored locally on the user’s device.

Users can delete this information by:

- Removing selected apps inside LockVault.
- Clearing LockVault’s storage through Android Settings.
- Uninstalling LockVault.

## Security

LockVault uses Android’s system security and authentication features to help protect access to the application and selected apps.

No security system can guarantee complete protection. Users should continue using a secure device screen lock and keep their Android software updated.

## Children’s Privacy

LockVault does not knowingly collect personal information from children or any other users.

## Changes to This Privacy Policy

This Privacy Policy may be updated if LockVault’s features or data practices change. Updates will be posted on this page with a revised effective date.
